# 訪談筆記範本(訪談當下照著記)

> 本檔為訪談紀錄格式的**正本(單一真實來源)**。`note-fill` 補完後的工作單、`lead-prep` 的輸入,格式都以本檔為準。
>
> **用途**:訪談當下快速記錄,記完即為「結構化訪談紀錄」,可直接丟進 `lead-prep` 彙整,不用再轉一手。
> **原則**:題號當指標,題目全文不用抄(訪綱裡已有);只記原話、小結、狀態標記。
> **適用**:測試人員與組長共用同一套格式,差別只在組長多一個【決策脈絡】標記(見下)。
>
> ⚠️ 若訪談當下只來得及記「題號 + 關鍵字」的速記、來不及記到本範本的完整度,改走 `note-fill` skill 喚醒記憶後,**人工**補成本範本格式,再進 `lead-prep`。

---

## 檔頭(每份筆記開頭填一次)

```
受訪者:____(稱呼,彙整時以此為準)
角色:□ 第一線測試人員  □ 測試組長
負責模組/產品:____
訪談日期:____
```

---

## 每題的記法(核心結構)

> 提醒:**原話**是後續根因分析的「實證」,**歸納**是你的「解讀」,兩者分開記。

```
[題號]  例:主題二-Q4
- 【原話】「____」  ← 盡量逐字,受訪者真的講的話
- 【歸納】____      ← 你自己整理的意思,跟原話分開
- 【狀態】____      ← 見下方標記說明
```

### 狀態標記怎麼選

| 標記 | 什麼時候用 |
|---|---|
| **【新增】** | 該位首次提到的內容(訪第 1 位時幾乎都是這個) |
| **【共識】** | 訪第 2 位時,確認跟第 1 位說法一致 |
| **【差異】** | 訪第 2 位時,跟第 1 位說法不同(記下差在哪,別硬合併) |
| **【決策脈絡】** | 僅組長用——解釋「當年為何這樣決定」,第一線給不了的料 |

---

## 兩個特殊情況的記法

**① 訪綱上沒有、當下臨時追問的題**
題號後面加「延伸追問」+ 那句問題,因為訪綱裡查不到:

```
主題二-Q4 延伸追問「那等的時候你都在幹嘛?」
- 【原話】「____」
- 【歸納】____
```

**② 訪第 2 位時的交叉確認**
直接標是針對第 1 位哪句話的確認:

```
主題二-Q4 交叉確認(對 A 的「轉站要等資訊組一整天」)
- 【原話】「____」
- 【狀態】【共識】 或 【差異】(差異點:____)
```

## 填寫範例

檔名:`小明_訪談紀錄.md`

```
受訪者:小明
角色:☑ 第一線測試人員
負責模組:訂單系統
訪談日期:2026/XX/XX

主題二-Q4 跨組依賴
- 【原話】「轉站都要等資訊組,有時候等一整天,我們自己又沒那個環境的權限」
- 【歸納】轉站等待是常態非偶發,卡在權限歸屬
- 【狀態】【新增】

主題三-Q7 過去自動化
- 【原話】「好像有人用 Selenium 寫過,後來那個人忙就沒人接」
- 【歸納】自動化曾推過但無人接手;小明非親身參與,屬聽聞
- 【狀態】【新增】(註:小明僅聽聞,細節待組長補)
```
